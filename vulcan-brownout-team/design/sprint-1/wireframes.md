# Vulcan Brownout: Sprint 1 Wireframes

## Overview

Sprint 1 wireframes focus on the **MVP user flows**: first-run experience, main list view, empty state, and error state. These wireframes are derived from the Sprint 0 work but scoped to Sprint 1 features only.

Key constraints:
- **No sorting controls** (implicit "critical first" sort only)
- **No pagination or infinite scroll** (full list renders; deferred to Sprint 2)
- **No threshold configuration panel** (hardcoded 15% for Sprint 1)
- **Manual refresh only** (no auto-refresh; WebSocket polling deferred to Sprint 2)

---

## Wireframe: Main Panel View (Desktop)

### Scenario
User has multiple battery-powered devices. Panel shows them ranked by urgency: critical devices at top (red), unavailable devices in middle (gray), healthy devices at bottom (green).

## Accessibility Notes

All wireframes follow WCAG 2.1 AA standards:

1. **Color contrast**: Text meets 4.5:1 ratio on all backgrounds
2. **Icon + text**: Status never conveyed by color alone (icons and text backup meaning)
3. **Touch targets**: Min 44px height (button size), min 48px for FAB
4. **Focus indicators**: Visible outline on keyboard navigation
5. **Semantic HTML**: Proper heading hierarchy, section roles
6. **Screen readers**: aria-labels for status, aria-live for dynamic updates
7. **Keyboard navigation**: Full panel usable via Tab, Enter, Space keys

## Summary

Sprint 1 wireframes are **clean, focused, and minimal**. They show the core MVP user flows: successful load (list view), empty state, and error state. All other features (sorting, filtering, threshold config, pagination) are removed from this sprint and scheduled for Sprint 2+.

This keeps the sprint focused and shippable.
